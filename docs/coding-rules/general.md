# 全般コーディングルール

## 言語

- TypeScript を使用する（`any` の使用は最小限に）
- 型定義は `interface` を優先し、ユニオン型が必要な場合は `type` を使用する

## フォーマット

- インデント: スペース2つ
- セミコロン: あり
- クォート: シングルクォート（JSX 内の属性はダブルクォート）
- 末尾カンマ: あり

## ファイル構成

- 関心の分離を意識する
  - ロジック: `.ts` / `.tsx`
  - スタイル: `.css`
  - 設定: `.json` / `.env`
- スタイルは CSS ファイルに集約し、コンポーネントファイルにスタイル定義を混在させない

## コメント

- 日本語で記述してよい
- 関数やモジュールの冒頭に目的を記述する
- 自明なコードにはコメントを付けない
- TODO / FIXME は積極的に残す

## エラーハンドリング

- 非同期処理は `try/catch` で囲む
- ユーザー向けのエラーは `alert` または UI で通知する
- 開発者向けのエラーは `console.error` / `console.warn` で出力する

## 環境変数

- クライアント側: `VITE_` プレフィックスを付ける
- デフォルト値を必ず設定する

```tsx
const SERVER_URL = import.meta.env.VITE_SERVER_URL || 'http://localhost:3001';
```

## Git

- コミットメッセージは日本語または英語で簡潔に
- 機能単位でコミットする
