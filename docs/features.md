# 機能仕様

## 実装済み機能

### 1. ビデオ通話機能

#### 基本機能
- **リアルタイムビデオ通話**: WebRTCによるP2P接続
- **複数人参加**: 同一ルームに複数ユーザーが参加可能
- **自動接続**: 新規参加者の自動検出と接続確立

#### ビデオ表示
- **グリッドレイアウト**: 参加者のビデオを自動配置
- **ミラーリング**: 自分のビデオは左右反転表示
- **ラベル表示**: 各ビデオに「You」「User 1」等のラベル

### 2. オーディオ機能

#### マイク制御
- **マイクミュート/アンミュート**: ワンクリックで切り替え
- **マイクデバイス選択**: 複数のマイクから選択可能
- **仮想デバイス除外**: Chrome対応のため、実際のマイクを自動選択
- **音声レベル可視化**: リアルタイムで音声入力を視覚化

#### スピーカー制御
- **スピーカーミュート/アンミュート**: 他の参加者の音声をミュート
- **スピーカーデバイス選択**: 出力デバイスの切り替え（Chrome/Edge対応）
- **エコー防止**: 自分の音声は常にミュート

### 3. ビデオ制御

- **カメラオン/オフ**: ビデオストリームの有効/無効切り替え
- **カメラデバイス選択**: （実装可能、現在は未実装）

### 4. ルーム管理

#### ルーム機能
- **Dashboard からの Room 作成**: ログイン済み管理者が Room を作成・管理
- **URL ベースの Room 参加**: `/room/:id` で直接参加（ログイン不要）
- **Room 存在チェック**: 存在しない Room ID へのアクセス時にエラー表示
- **参加用 URL 共有**: Dashboard から Room の参加用 URL をクリップボードにコピー
- **Room 削除**: 所有者のみ削除可能（確認ダイアログ付き）
- **参加者数表示**: 現在の接続ユーザー数を表示
- **退出機能**: ルームから退出してホーム画面に戻る
- **退出確認ダイアログ**: 退出ボタン押下時に確認モーダルを表示し、誤操作による退出を防止

#### 接続管理
- **自動再接続**: （Socket.IOのデフォルト機能）
- **切断検知**: ユーザー退出時の自動クリーンアップ

### 4.5. Google OAuth 認証

#### 認証機能
- **Google ログイン**: `@react-oauth/google` によるポップアップ型 Google Sign-In
- **JWT 認証**: サーバー側で Google ID トークンを検証し、アプリ固有の JWT を発行（有効期限 7 日）
- **認証状態の永続化**: JWT を localStorage に保存し、ページリロード後も認証状態を維持
- **ログアウト**: Dashboard からワンクリックでログアウト

#### ユーザーの 2 パターン
- **Room 管理者**: Google OAuth でログインし、Room の作成・管理・URL 共有を行う
- **Room 利用者**: ログイン不要。共有された URL から直接 Room に参加

#### ルーティング
- `/` : ランディングページ（Google ログインボタン）。認証済みなら `/dashboard` にリダイレクト
- `/dashboard` : Room 管理画面（ログイン必須）
- `/room/:id` : Room 参加画面（認証不要）

#### Room データ永続化
- JSON ファイルベース（`server/data/rooms.json`）
- 拡張可能な `settings` フィールドで将来のセキュア Room 等に対応

### 5. 画面共有

- **画面共有開始/停止**: `getDisplayMedia()` によるデスクトップ/ウィンドウの共有
- **共有画面の表示**: 画面共有中は専用のビューで大きく表示
- **ブラウザ停止ボタン対応**: ブラウザの「共有を停止」ボタンで自動停止
- **排他制御**: 他の参加者が画面共有中は共有ボタンが無効化
- **DataChannel通知**: 画面共有の開始/停止を全Peerに通知

### 6. UI/UX

#### デザイン
- **モダンなUI**: ダークテーマのクリーンなデザイン
- **アイコンベース**: Lucide Reactによる直感的なアイコン
- **レスポンシブ**: 画面サイズに応じた自動調整

#### コントロールバー
- **マイクコントロール**: アイコン + デバイス選択ドロップダウン
- **ビデオコントロール**: カメラオン/オフ切り替え
- **スピーカーコントロール**: アイコン + デバイス選択ドロップダウン
- **退出ボタン**: 目立つ赤色のボタン

#### 音声可視化
- **リアルタイム波形**: Web Audio APIによる音声レベル表示
- **視覚的フィードバック**: 音声入力時に円が拡大

### 6. デバイス管理

#### 対応ブラウザ
- **Chrome/Edge**: 完全対応（マイク、スピーカー選択含む）
- **Firefox**: 基本機能対応（スピーカー選択は制限あり）
- **Safari**: 基本機能対応（一部制限あり）

#### デバイス検出
- **自動列挙**: 利用可能なデバイスを自動検出
- **ラベル表示**: デバイス名を表示
- **権限管理**: ブラウザのネイティブ権限ダイアログ

## 技術的な実装詳細

### WebRTC接続

#### Peer接続タイプ
- **Initiator (発信側)**: 既存ユーザーが新規ユーザーに接続
- **Receiver (受信側)**: 新規ユーザーが既存ユーザーからの接続を受ける

#### シグナリング
- **統一シグナルパス**: Offer/Answer/ICE Candidateを単一のイベントで処理
- **Trickle ICE**: ICE Candidateの段階的送信

#### STUN/TURNサーバー
- **STUN（プライマリ）**: `stun.l.google.com:19302`（Google公開STUNサーバー）
- **STUN（セカンダリ）**: `stun.mynt.work:3478`（自前STUNサーバー）
- **TURN**: `stun.mynt.work:3478`（P2P接続不可時のメディアリレー）
- **TURNS**: `stun.mynt.work:5349`（TLS経由のTURN接続）
- **シグナリングサーバー**: `sock.mynt.work`（Socket.IOによるWebRTCシグナリング）

> GoogleのSTUNで先にアドレス解決を試み、NAT越えできない場合は自前のTURNにフォールバックする構成。

### オーディオ処理

#### AudioContext
- **クローントラック**: Chrome対応のため、音声トラックをクローン
- **AnalyserNode**: FFT解析による音声レベル取得
- **自動再開**: ユーザー操作時にAudioContextを再開

#### デバイス切り替え
- **Track置換**: `replaceTrack()`による動的なトラック変更
- **setSinkId()**: 出力デバイスの動的変更（Chrome/Edge）

### 状態管理

#### React State
- `localStream`: ローカルメディアストリーム
- `peers`: 接続中のPeerリスト
- `isAudioEnabled`: マイクの有効/無効
- `isVideoEnabled`: カメラの有効/無効
- `isSpeakerEnabled`: スピーカーの有効/無効
- `audioDevices`: 利用可能なマイクデバイス
- `audioOutputDevices`: 利用可能なスピーカーデバイス

#### Refs
- `socketRef`: Socket.IO接続の参照
- `peersRef`: Peerオブジェクトの参照（Socket IDをキー）
- `localStreamRef`: ローカルストリームの参照

## パフォーマンス

### 推奨環境
- **CPU**: 2コア以上
- **メモリ**: 4GB以上
- **ネットワーク**: 上り/下り 1Mbps以上（参加者1人あたり）
- **ブラウザ**: 最新版のChrome、Firefox、Edge

### 制限事項
- **最大参加者数**: 6人程度（P2P接続の制限）
- **ビデオ解像度**: ブラウザとデバイスに依存
- **フレームレート**: 通常30fps、帯域により自動調整
